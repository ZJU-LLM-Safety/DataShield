# Security Policy

DataShield is a research release for controlled fine-tuning safety evaluation. Some data and generated artifacts may be safety-sensitive.

## Reporting

Please do not post safety-sensitive details, exploit instructions, or harmful dataset excerpts in public issues. For security or misuse concerns, use GitHub private vulnerability reporting if it is enabled for the repository, or contact the repository maintainers through a private channel.

For non-sensitive bugs, documentation issues, or reproducibility questions, open a normal GitHub issue.

## Scope

Reports are especially helpful when they involve:

- accidental exposure of non-anonymous author or affiliation metadata;
- unsafe redistribution or misuse risks in released data;
- model artifacts, scripts, or examples that enable harmful behavior beyond controlled evaluation;
- dependency or execution issues that could compromise a user's environment.
